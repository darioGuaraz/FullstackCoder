Introducción al desarrollo web

Esta estructura corresponde a una práctica introductoria donde se pone en uso el lenguaje HTML junto a CSS. A través de este trabajo se busca entender la importancia de estructurar correctamente el contenido usando etiquetas semánticas, que permiten que el navegador, los motores de búsqueda y las tecnologías de asistencia comprendan mejor la información del sitio.

Para comenzar a aprender HTML es fundamental conocer las etiquetas más comunes, su función y cómo organizarlas de forma coherente. Todo documento HTML inicia con <!DOCTYPE html>, seguido por la estructura básica: <html>, dentro de la cual se colocan las secciones <head> (encabezado del documento) y <body> (contenido visible).

A lo largo del documento, se utilizan etiquetas semánticas que ayudan a darle sentido a cada bloque. A continuación, explicaremos algunas de las más importantes.

Fundamentos de HTML

¿Qué es HTML y por qué es importante?

HTML (HyperText Markup Language) es el lenguaje que define la estructura del contenido web. Usamos etiquetas como <section>, <article>, <header>, <footer> y <nav> para dar un significado claro a cada parte del sitio, facilitando la accesibilidad, el mantenimiento del código y el posicionamiento en buscadores (SEO).

¿Para qué sirven las etiquetas semánticas?

Las etiquetas semánticas indican qué tipo de contenido contienen. Por ejemplo:

  • <header>: Encabezado del sitio o sección.
  • <nav>: Barra de navegación.
  • <main>: Contenido principal del sitio.
  • <section>: Sección temática de contenido.
  • <article>: Contenido independiente o autoexplicativo.
  • <aside>: Contenido complementario.
  • <footer>: Pie de página.

Buenas prácticas en HTML

Una buena práctica en HTML es separar el contenido (HTML) de la presentación (CSS). De esta manera, el código es más limpio, escalable y fácil de leer. Además, utilizar nombres de clases descriptivos y etiquetas correctas mejora la comprensión tanto para otros desarrolladores como para los usuarios.

Introducción a CSS

¿Qué es CSS y para qué se utiliza?

CSS (Cascading Style Sheets) es el lenguaje encargado del diseño y la apariencia visual del sitio. Permite modificar colores, tipografías, márgenes, ubicaciones y animaciones, logrando una experiencia de usuario atractiva y coherente con la identidad del proyecto.

Buenas prácticas en CSS

Algunas buenas prácticas en CSS incluyen el uso de selectores claros, reutilización de clases, organización del código en secciones y el uso de propiedades como flex o grid para lograr diseños responsivos adaptables a distintos dispositivos.

  • Diseño adaptado a múltiples dispositivos (responsivo)
  • Separación de estructura y estilo
  • Código accesible, ordenado y escalable

Aclaración: el diseño actual no es responsive. Se recomienda visualizar el sitio en computadoras de escritorio para una mejor experiencia.

Formulario y accesibilidad

El formulario permite al usuario enviar datos a través de entradas como input o textarea. Es esencial acompañar estos elementos con etiquetas label asociadas, ya que mejora la accesibilidad y la usabilidad del sitio. También se pueden añadir atributos como placeholder, required o validaciones para mejorar la interacción.

Contenido complementario

El elemento <aside> se utiliza para contenido complementario al principal, como noticias, enlaces, anuncios o recursos externos. Aunque no es el foco principal del sitio, mejora el contexto general y la navegación del usuario.

Aprender HTML paso a paso implica entender para qué sirve cada una de estas etiquetas, cómo estructurar la información y cómo combinarlo con CSS para lograr sitios más atractivos y funcionales. Lo ideal es practicar con ejemplos reales, como este, donde se pone en uso cada parte de una página web.
